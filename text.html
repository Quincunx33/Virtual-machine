Help & Guides

1. Creating a Blank Disk Image

Use this method when you want to install an OS from an .iso file. This creates an empty virtual hard drive which you can upload in the "Install an OS" dialog.

Linux / macOS / Termux

Use the dd command to create a file filled with zeros.

dd if=/dev/zero of=blank.img bs=1M count=<size_in_mb>

Example (for 1GB):

dd if=/dev/zero of=disk.img bs=1M count=1024
Windows

Use fsutil in Command Prompt or PowerShell (run as Administrator).

fsutil file createnew blank.img <size_in_bytes>

Example (for 1GB): fsutil file createnew disk.img 1073741824

Note: Size must be in bytes (1GB = 1073741824 bytes).

2. Creating a Bootable Disk Image

Use this method to prepare an image for the "Boot Disk Image" option. This involves installing an OS onto the disk image using a local emulator like QEMU.

Step 1: Install QEMU. It's a powerful, free command-line emulator.

QEMU Installation

macOS (Homebrew): brew install qemu

Ubuntu/Debian: sudo apt-get install qemu-system-x86

Windows: Download from the QEMU for Windows site.

Step 2: Create a blank disk image. For this emulator, a raw .img is required.

Create Image File
qemu-img create -f raw my-os-disk.img 2G

This creates a 2-gigabyte raw disk image named my-os-disk.img.

Step 3: Install the OS. Run QEMU, booting from your OS installer .iso and attaching the disk image you just created.

Run Installer in QEMU
qemu-system-i386 -cdrom /path/to/os_installer.iso -drive file=my-os-disk.img,format=raw -boot d -m 512
  • -cdrom ...: Path to your installer (e.g., a Windows 98 ISO).
  • -drive file=...: The disk image to install onto.
  • -boot d: Boot from the CD-ROM drive first.
  • -m 512: Allocate 512 MB of RAM to the QEMU VM.

Step 4: Use the image. After the installation finishes inside QEMU, shut down the QEMU VM. Your my-os-disk.img file is now a bootable disk image ready to be uploaded here!
